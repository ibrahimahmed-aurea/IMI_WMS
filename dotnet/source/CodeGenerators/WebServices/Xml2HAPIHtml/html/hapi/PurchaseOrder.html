Purchase Order Line

Purchase Order Line
Specification of the planned reception.
Includes information of the product, planned quantity and final destination of the received goods.

Column Method Type Mand. Note
ClientIdentity N, C, R c17 * The identity of the Client.
PurchaseOrderNumber N, C, R c35 * The identity of the Purchase Order.
PurchaseOrderSequence N, C, R n3 * When there are multiple occurrences of the Purchase Order in the warehouse system this tells in which order they have been received from the host system.
PurchaseOrderLinePosition N, C, R n4 * The identity of the Purchase Order Line.
PurchaseOrderLineSequence N, C, R n3 * Used as an extra identity of the Purchase Order Line.
OwnerIdentity N, C c35 N The identity of the Owner.
ProductNumber N, C c35 N The identity of the Product.
ProductDescription N, C c35 - The description/name of the Product. It is displayed in the dialogs for the ASN Line. Could be another Product Description than that in the Product table.
VendorProductNumber N, C c35 - The Vendors identity of the Product.
ScheduledArrivalDateTime N, C date N Expected Arrival Date and Time.
PackageIdentity N, C c17 N The identity of the Lowest Unit of Measure / Base Package of the Product that can be handled.
PurchaseQuantity N, C n20,6 N The ordered quantity of the product.
ProductionLotIdentity N, C c40 - The Manufacturing Lot for the received goods.
ProductionSubLotIdentity N, C c40 - The Manufacturing Sublot for the received goods.
MarketingLotIdentity N, C c20 - The Marketing Lot for the received goods.
PurchasePrice N, C n22,7 - Purchase price per Lowest Unit of Measure (Base Package) of the Product.
ShipToCustomerQualifier N, C c3 - Type of Party that is the Ship-to Customer.
ShipToCustomerIdentity N, C c35 - The Party that finally will receive the goods.
ShipToPartyQualifier N, C c3 - The type of Party that is the Ship-to Party.
ShipToPartyIdentity N, C c35 - The identity of the Party that will receive the goods when they are shipped from the Ship-from Party. So in this case the Ship-to Party is your own warehouse.
CustomerOrderNumber N, C c35 - The identity of the Customer Order. Used for transit/warehouse transfer goods.
CustomerOrderSequence N, C n3 - A Sequence Number used by the Host system to make it possible to send a Customer Order more then once (e.g. backorder). Used for transit/warehouse transfer goods.
CustomerOrderLinePosition N, C n5 - The identity of the Customer Order Line. Used for transit/warehouse transfer goods.
CustomerOrderLineKitPosition N, C n2 - The Customer Order Line contains an Assembly Product. Used for transit/warehouse transfer goods.
CustomerOrderLineSequence N, C n3 - Used as an extra identity of the Customer Order Line. Used for transit/warehouse transfer goods.
InventoryStatusCode N, C c8 - A Hold Code may be applied on an Item Load or a Pick Location to temporarily exclude it from further processing.
InventoryStatusDays N, C n5 - Number of Hold Days
Instructions N, C c400 - Free text for the ASN Line.
DeliveryWindowFirst N, C date - First possible Arrival Date and Time.
DeliveryWindowLast N, C date - Last possible Arrival Date and Time.
OriginalOrderIdentity N, C c35 - The final recipients Purchase Order Number reference. Used for transit goods.
OriginalOrderLinePosition N, C n16 - The final recipients Purchase Order Line Number reference. Used for transit goods.
OriginalCustomerReference N, C c35 - The final recipients Customer Reference. Used for transit goods.
SellToCustomerQualifier N, C c3 - Type of Party that is the Sell-to Customer.
SellToCustomerIdentity N, C c35 - The Customer that made the Order according to the Customer Order.
LastRow N, C, R c255 - n/a
QualityLotIdentity N, C c20 - Can be used for optional classification of the stored goods in the warehouse (quality, colour, etc).
QuantityUpperTolerance N, C n20,6 - The maximum quantity (number of Lowest Units of Measure) that can be received in this Inbound Order Line. This quantity is transferred from the Host System.
CatchMeasureLowerTolerance N, C n20,6 - The minimum accepted Measured Quantity per ordered unit. It is used to check that the registered Measured Quantity is within the limits of reason. This tolerance quantity is transferred from the Host System.
Example: The Ordered Quantity is 20 cheeses of 700 gram each. The Lower Limit for Measured Quantity = 550 gram. When the picker measures all 20 cheeses, the system will accept any registered weight that is above 11 kg (=20x550g).
CatchMeasureUpperTolerance N, C n20,6 - The maximum accepted Measured Quantity per ordered unit. It is used to check that the registered Measured Quantity is within the limits of reason. This tolerance quantity is transferred from the Host System.
Example: The Ordered Quantity is 20 cheeses of 700 gram each. The Upper Limit for Measured Quantity = 770 gram. When the picker measures all 20 cheeses, the system will accept any registered weight that is below 15.4 kg (=20x770g).
DeltaPurchaseQuantity C n20,6 - The ordered quantity of the product.
