Column Method Type Mand. Note

CustomerOrderLineKitPosition N n2 * The Customer Order Line contains an Assembly Product.

N = Normal

RequestedShipdate N date * The date that the customer has requested that the goods shall be shipped from the warehouse.
AcknowledgedShipdate N date * The shipping date that the Warehouse System has acknowledged to the customer.

RequestedDeliveryDate N date * Requested Arrival Date.
AcknowledgedDeliveryDate N date * Acknowledged Arrival Date.
WarehouseIdentity N c4 * The identity of the Warehouse.
ProductNumber N c35 * The identity of the Product.
OwnerIdentity N c35 * The identity of the Owner.
ProductionLotRequireMent N c1 - Controls if there is check of the Manufacturing Lot when selecting the goods/Item Load :
<null> = No control
1 = Any Lot (must belong to a Lot)
2 = Only one Lot (must belong to the same Manufacturing Lot)
3 = Specified Lot (must belong to the stated Manufacturing Lot)
MarketingLotIdentity N c20 - The Marketing Lot that the picked goods have to belong to.
QualityLotIdentity N c20 - Can be used for optional classification of the stored goods in the warehouse (quality, colour, etc).
PackageIdentity N c17 * The identity of the Unit of Measure / Package.
OrderedQuantity N n20,6 - The ordered quantity of the product.
OrderedQuantityTolerancePlus N n20,6 - The number of Lowest UMs that the ordered quantity in a Pick Order Line can differ by upwards, so that this package/unit may still be picked without having to register a discrepancy code.
This value is normally defined for each Package of the Product but that value is always overridden if there is a value transferred from the host system in the Customer Order Line.
OrderedQuantityToleranceMinus N n20,6 - The number of Lowest UMs that the ordered quantity in a Pick Order Line can differ by downwards, so that this package/unit may still be picked without having to register a discrepancy code.
This value is normally defined for each Package of the Product but that value is always overridden if there is a value transferred from the host system in the Customer Order Line.
KitDetailQuantity N n20,6 - The Quantity of the Component Product that is needed for each copy of the Kit Product.
RequestedShelfLife N n4 - Not used.
ForcePackage N c1 - Forced Package.
SerialNumberHandling N c1 - Controls (1=Yes,0=No) if the Serial Number is required at Case Pick.
SerialNumber N c21 - A unique identity for a single Item Load. Only used for Case Pick and Serial Number handling!
VATPercent N n6,4 - The financial conditions for the Customer Order Line, which VAT applies.
DepartureIdentity N c255 - n/a
DepartureWeekday N c255 - n/a
DepartureTime N date - n/a
DeliveryMethod N c17 - The identity of method that will be used when shipping the goods from the warehouse.
Measure N c1 - Controls (1=Yes,0=No) if the Products shall go through Catch Measure (Quantity check) at Case Pick.
MeasureQualifier N c4 - States in which unit to measure the goods if Catch Measure is set.
SalesPrice N n17,7 - Price for the Product per Package (info).
LineTotal N n17,7 - Price for the Product per Package (info).
ShipToCustomerProductNumber N c35 - The Product Number at the Customer site.
PurchaseOrderNumber N c35 - The identity of the Purchase Order.
PurchaseOrderSequence N n3 - The Sequence Number of the Purchase Order that the Customer Order Line is connected to.
PurchaseOrderLinePosition N n4 - The identity of the Purchase Order Line.
PurchaseOrderLineSequence N n3 - Used as an extra identity of the Purchase Order Line.
OriginalOrderIdentity N c35 - The final recipients Purchase Order Number reference. Used for transit goods.
OriginalOrderLinePosition N n16 - The final recipients Purchase Order Line Number reference. Used for transit goods.
DepartureLoadIdentity N c35 - The identity of the Transport Load, that can be a Customer Order (whole or part of) or a Transit Consignment in a Departure.
PlanLoad N c1 - Indicates (1=Yes,0=No) if the Transport Load should be connected to a Departure.
DateSelectionType N c1 - Indicates how the date/time fields are evaluated.
Values:
S = Ship Date decided from Earliest and Latest Ship Date and Time
D = Delivery Date decided from Earliest and Latest Delivery Date and Time
Default value is Ship Date.
FirstPlannedDeliveryDtm N date - The earliest delivery date and time at Customer site. Valid if Date Selection Type = Delivery.
This date is mandatory if the Date Selection Type = Delivery and if the Transport Loads should be connected to a departure.
LastPlannedDeliveryDtm N date - The latest delivery date and time at Customer site. Valid if Date Selection Type = Delivery.This date is mandatory if the Date Selection Type = Delivery and if the Transport Loads should not be connected to a departure.
FirstPlannedShipDtm N date - The earliest date and time for shipping the goods from the Warehouse. Only used when the Date Selection Type = Ship.
This date is mandatory if the Date Selection Type = Ship and if the Transport Loads should be connected to a departure.
LastPlannedShipDtm N date - The latest date and time for shipping the goods from the Warehouse. Only used when the Date Selection Type = Ship.
This date is mandatory if the Date Selection Type = Ship and if the Transport Loads should not be connected to a departure.
OwnerIdentityAtShipToCustomer N c35 - The identity of the Owner that will registered for the goods at the Final Node (the Ship-to Customer).
InventoryStatusCode N c8 - A Hold Code may be applied on an Item Load or a Pick Location to temporarily exclude it from further processing.
OrderedQuantityLowerTolerance N n20,6 - The minimum quantity (number of Lowest Units of Measure) that has to be picked for this Order Line. This quantity is transferred from the Host System.
OrderedQuantityUpperTolerance N n20,6 - The maximum quantity (number of Lowest Units of Measure) that can be picked for this Order Line. This quantity is transferred from the Host System.
CatchMeasureLowerTolerance N n20,6 - The minimum accepted Measured Quantity per ordered unit. It is used to check that the registered Measured Quantity is within the limits of reason. This tolerance quantity is transferred from the Host System.
Example: The Ordered Quantity is 20 cheeses of 700 gram each. The Lower Limit for Measured Quantity = 550 gram. When the picker measures all 20 cheeses, the system will accept any registered weight that is above 11 kg (=20x550g).
CatchMeasureUpperTolerance N n20,6 - The maximum accepted Measured Quantity per ordered unit. It is used to check that the registered Measured Quantity is within the limits of reason. This tolerance quantity is transferred from the Host System.
Example: The Ordered Quantity is 20 cheeses of 700 gram each. The Upper Limit for Measured Quantity = 770 gram. When the picker measures all 20 cheeses, the system will accept any registered weight that is below 15.4 kg (=20x770g).
ProductionLotIdentity N c40 - The Manufacturing Lot that the picked goods have to belong to (if required by Manufacturing Lor Requirement).
ProductionSubLotIdentity N c40 - The Manufacturing Sublot that the picked goods have to belong to (if required by Manufacuring Lot Requirement).
