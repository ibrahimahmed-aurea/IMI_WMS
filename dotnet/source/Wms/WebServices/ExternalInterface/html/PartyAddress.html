Segment PartyAddress

Description:

Party
Updates data for a Party.

Attributes:

TagMethodTypeOracle TypeMandatoryNote
OPCODEN, C, RstringVARCHAR2(1) Operation Code
0 - No Operation
1 - New
2 - Change
3 - Remove
Note! An operation code is only useful if at least one tag supports the method
PartyQualifierN, C, RstringVARCHAR2(3)N, C, RParty Type
The type of the Party:
CA = Forwarder (NOT Client dependent)
CL = Client (NOT Client dependent)
CU = Customer (Client dependent)
GO = Owner (Client dependent)
OT = Other (NOT Client dependent)
SU = Vendor (Client dependent)
WH = Warehouse (NOT Client dependent)
CD = Customs Declarant (NOT Client dependent)
PartyIdentityN, C, RstringVARCHAR2(35)N, C, RParty Identity
The identity of the Party.
PartyProfileN, CstringVARCHAR2(35) Party Profile
The Party Type (Owner, Customer, Vendor).
If the Party Type is client dependent then this is
a foreign key to the table Partyqualifer = Table.Column accordingly:
Owner = OWN.OWNID
Customer = CUS.CUSID
Vendor = SUP.SUPID
PartyStatusN, CstringVARCHAR2(1) Party Status
The status of the Party:
N = Normal
T = Temporary

The Name of the party.

Line 5 of the name for the party.

O = Other

PayCustomsDeclN, CstringVARCHAR2(1) Pay Customs Declaration
The Customs Declaration controls (1=Yes,0=No) if the Customs Declaration shall be paid or not.
