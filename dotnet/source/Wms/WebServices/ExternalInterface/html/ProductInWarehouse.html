Segment ProductInWarehouse

Description:

Product in Warehouse
Handling rules for a product in a specific warehouse.

Attributes:

TagMethodTypeOracle TypeMandatoryNote
OPCODEN, C, RstringVARCHAR2(1) Operation Code
0 - No Operation
1 - New
2 - Change
3 - Remove
Note! An operation code is only useful if at least one tag supports the method
ProductNumberN, C, RstringVARCHAR2(35)N, C, RProduct Number
The identity of the Product.
WarehouseIdentityN, C, RstringVARCHAR2(4)N, C, RWarehouse Identity
The identity of the Warehouse.
InventoryAuditFrequencyN, CNullableNUMBER(4) Interval between Audits for Item Load (days)
The number of days that has to pass from a previous inventory audit before the system
is allowed to generate a new inventory audit order for the product's Item Loads.
StockedProductN, CstringVARCHAR2(1) Stocked Product
Controls (1=Yes,0=No) if this is a Stocked Product.
If it is not a Stocked Product (=No), then this might be Transit Goods, Information or a Service Product.
OwnerIdentityN, CstringVARCHAR2(35) Owner Identity
The identity of the Owner.
StorageTemperatureMinN, CNullableNUMBER(4) Min Temp
The minimum storage temperature controls in which Temperature Class the goods will be stored.
StorageTemperatureMaxN, CNullableNUMBER(4) Max Temp
The maximum storage temperature controls in which Temperature Class the goods will be stored.
StorageTemperatureQualifierN, CstringVARCHAR2(1) Unit of Measurement on Temperature
Only info, no logic implemented.
FrequencyN, CNullableNUMBER(8) Product Handling Frequency
The Handling Frequency (rate of turnover in the Warehouse) controls
in which Handling Frequency Class the goods will be stored.
InstructionsN, CstringVARCHAR2(800) Instructions
Free text.
HygieneCheckN, CstringVARCHAR2(1) Hygiene Check
Not used.
TemperatureCheckN, CstringVARCHAR2(1) Temperature Check
Controls (1=Yes,0=No) if the received goods containing the Products shall have a Temperature Check.
RefuseTempHighN, CNullableNUMBER(7,2) Max Temperature
If the temperature of the Vehicle Load Unit (VLU) or the received goods
are above this limit, the user may reject the VLU or the goods.
RefuseTempLowN, CNullableNUMBER(7,2) Min Temperature
If the temperature of the Vehicle Load Unit (VLU) or the received goods
are below this limit, the user may reject the VLU or the goods.
WarnTempHighN, CNullableNUMBER(7,2) Upper Warning Temperature Limit
Not used.
WarnTempLowN, CNullableNUMBER(7,2) Lower Warning Temperature Limit
Not used.
PackageChangeN, CstringVARCHAR2(1) Enter Lowest Unit of Measure
When the user has to register the Lowest Unit of Measure for the received goods containing the Product:
- Never
- During Package Change (the Lowest Unit of Measure must be registered until the first time the new Package is received)
- Always
ProductionLotHandlingN, CstringVARCHAR2(1) Manufacturing Lot at Outbound
Controls (1=Yes,0=No) if the Manufacturing Lot is used when generating Pick Order Lines.
StorageLotHandlingN, CstringVARCHAR2(1) Quality Lot at Outbound
Controls (1=Yes,0=No) if the Quality Lot is used when generating Pick Order Lines.
ProductionLotHandling_InN, CstringVARCHAR2(1) Manufacturing Lot at Inbound
Controls (1=Yes,0=No) if the Manufacturing Lot has to be registered at receiving.
MarketingLotHandling_InN, CstringVARCHAR2(1) Marketing Lot at Inbound
Controls (1=Yes,0=No) if the Marketing Lot has to be registered at the receiving.
StorageLotHandling_InN, CstringVARCHAR2(1) Quality Lot at Inbound
Controls (1=Yes,0=No) if the Quality Lot has to be registered at the receiving.
ProductionLotVerifyN, CstringVARCHAR2(1) Manufacturing Lot Required
Controls (1=Yes,0=No) if the Manufacturing Lot has to be registered at Pick.
