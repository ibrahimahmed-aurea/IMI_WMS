Segment CustomerOrderHead

Description:

Customer Order

Attributes:

TagMethodTypeOracle TypeMandatoryNote
OPCODEN, C, RstringVARCHAR2(1) Operation Code
0 - No Operation
1 - New
2 - Change
3 - Remove
Note! An operation code is only useful if at least one tag supports the method
CustomerOrderNumberNstringVARCHAR2(35)NCustomer Order Number
The identity of the Customer Order.
CustomerOrderSequenceNNullableNUMBER(3)NCustomer Order Sequence Number
A Sequence Number used by the Host system to make it possible
to send a Customer Order more than once (e.g. backorder).
CustomerOrderTypeNstringVARCHAR2(2)NCustomer Order Type
The type of the Customer Order. Comes from the Host System and does not control any logic
in the Warehouse except for what is described below:
- LO = Local Order inhibits sending of Order-related information via
the HAPI and EDI interface, such as Pick Result and Receipt Report.
- Other values are just information in the Warehouse system.
CustomerOrderClassNstringVARCHAR2(2)NCustom Order Type Identity
The type of Customer Order:
N = Normal
TR = Transit / Stock Replenishment
Installation dependent. Controls e.g. choice of Method of Shipment.
CustomerOrderReferenceNstringVARCHAR2(35) Reference
A Reference from the Customer.
BillToCustomerIdentityNstringVARCHAR2(35) Bill-to Customer
The Party that will be invoiced for the ordered goods.
SellToCustomerIdentityNstringVARCHAR2(35)NSell-to Customer Identity
The Customer that made the Order according to the Customer Order.
ShipToCustomerIdentityNstringVARCHAR2(35)NShip-to Customer Identity
The Party that finally will receive the goods.
PickPriorityNNullableNUMBER(2)NPick Priority
The priority (1-9, where 1 is highest) is set to the Pick Orders that are generated.
This priority can, if required, be changed manually while picking.
DispatchFeeNNullableNUMBER(17,7) Service Charge
Costs for Service Charge.
CODFeeNNullableNUMBER(17,7) COD Fee
Cash on Delivery Fee.
FreightCostNNullableNUMBER(17,7) Freight Cost
The cost for transporting the goods to the Ship-to Customer.
FreightPartyQualifierNstringVARCHAR2(3) Forwarder's Party Type
The Type of Party that has a forwarder agreement with the Forwarder.
FreightPartyIdentityNstringVARCHAR2(35) Forwarder's Party Identity
The identity of the Party that has a forwarder agreement with the Forwarder.
DivisionNstringVARCHAR2(17) Division
Division, department, section, workshop or similar
subordinate unit of a company / organization.
TermsOfDeliveryNstringVARCHAR2(6) Terms of Shipment
The Terms of Shipment for the Consignment stated
as a code according to Combiterm 2000.
TermsOfDeliveryLocationNstringVARCHAR2(35) Terms of Shipment Destination
The destination for the goods according to the Terms of Shipment.
Received from the external Carrier Compliancy Provider.
CODReferenceNstringVARCHAR2(35) Reference for Cash-on-Delivery
Reference for the Cash-on-Delivery, usually a bank or giro account for the Bill-to Customer.
