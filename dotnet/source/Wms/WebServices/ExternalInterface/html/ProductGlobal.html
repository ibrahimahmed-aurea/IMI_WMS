Segment ProductGlobal

Description:

Product
Register basic data about a product.

Attributes:

TagMethodTypeOracle TypeMandatoryNote
OPCODEN, C, RstringVARCHAR2(1) Operation Code
0 - No Operation
1 - New
2 - Change
3 - Remove
Note! An operation code is only useful if at least one tag supports the method
ProductNumberN, C, RstringVARCHAR2(35)N, C, RProduct Number
The identity of the Product.
ProductDescription1N, CstringVARCHAR2(256)NProduct Description
The description / name of the Product.
ProductDescription2N, CstringVARCHAR2(256) Product Description
The extra description / name of the Product.
ProductStockGroupIdentityN, CstringVARCHAR2(12) Product Stock Group
The identity of the Product Stock Group.
Contains the handling rules for how receiving, shipping, etc. occur
for products included in the product stock group.
ProductTypeN, CstringVARCHAR2(2) Product Type Identity
The identity of the Product Type. With Product Types you can categorize / group
all the products into more manageable groups internally in the Warehouse.
ProductDateTypeN, CstringVARCHAR2(1) Product Date Type
The type of Product Date that is used:
+ = Expiry Date
- = Manufacturing Date
= = Arrival Date
0 = No Date
ProductShipDateLeadTimeN, CNullableNUMBER(4) Last Ship Date
The number of days between the Last Ship Date and the Expiry Date for the Product.
ProductDeliveryDateLeadTimeN, CNullableNUMBER(4) Arrival Date
The number of days between the Arrival Date and the Expiry Date for the Product.
ProductManufacturDateLeadTimeN, CNullableNUMBER(4) Manufacturing Date
The number of days between the Manufacturing Date and the Expiry Date for the Product.
ProductionLotHandlingN, CstringVARCHAR2(1) Manufacturing Lot at Outbound
Controls (1=Yes,0=No) if the Manufacturing Lot is used when generating Pick Order Lines.
StorageLotHandlingN, CstringVARCHAR2(1) Quality Lot at Outbound
Controls (1=Yes,0=No) if the Quality Lot is used when generating Pick Order Lines.
SerialNumberHandlingN, CstringVARCHAR2(1) Serial Number
Controls (1=Yes,0=No) if the Serial Number is required at Case Pick.
FIFOAllowedDiscrepancyN, CNullableNUMBER(4) Number of days (FIFO)
States the maximum number of days that the FIFO date may differ among the picked item loads.
Shipping from the Warehouse normally strictly occurs according to FIFO (First In First Out),
but this can lead to extra long movements to drive around and pick-up the oldest item loads.
To avoid this, the FIFO date for picked item loads may differ from the oldest FIFO date. Examples:
If FIFOPLUS = 0 then picking occurs strictly according to FIFO.
If FIFOPLUS = 20 then FIFO can differ with 20 days from the oldest Item Load that is picked.
InstructionsN, CstringVARCHAR2(800) Instructions
Free text.
ProductSaleableN, CstringVARCHAR2(35) Stocked Product Number
If the Packing Material Types also can exist as a saleable product it has a Product Number
as stocked product. Otherwise this is empty <null>.
ProductionLotHandling_InN, CstringVARCHAR2(1) Manufacturing Lot at Inbound
Controls (1=Yes,0=No) if the Manufacturing Lot has to be registered at receiving.
MarketingLotHandling_InN, CstringVARCHAR2(1) Marketing Lot at Inbound
Controls (1=Yes,0=No) if the Marketing Lot has to be registered at the receiving.
StorageLotHandling_InN, CstringVARCHAR2(1) Quality Lot at Inbound
Controls (1=Yes,0=No) if the Quality Lot has to be registered at the receiving.
ProductionLotVerifyN, CstringVARCHAR2(1) Manufacturing Lot Required
Controls (1=Yes,0=No) if the Manufacturing Lot has to be registered at Pick.
ProductTransportIdentityN, CstringVARCHAR2(5) Transport Condition Identity
Gives the conditions (normal, freeze, cold, dry, non-vibrating etc.)
for the vehicle that transports the goods / product.
CatchMeasureN, CstringVARCHAR2(1) Catch Measure
Controls (1=Yes,0=No) if the Products shall go through Catch Measure (Quantity check).
CatchMeasureQualifierN, CstringVARCHAR2(4) Catch Measure Unit
States in which unit to measure the goods if Catch Measure is set.
CatchMeasureRcvN, CstringVARCHAR2(1) Catch Measure
Controls (1=Yes,0=No) if the received goods containing the Products
shall go through Catch Measure (Quantity check).
CatchMeasureQualifierRcvN, CstringVARCHAR2(4) Catch Measure Unit
States in which unit the goods were measured during the reception process.
HazardousGoodsN, CstringVARCHAR2(1) Hazardous Goods
Controls (1=Yes,0=No) if the Product is considered as Hazardous Goods.
Instructions_InN, CstringVARCHAR2(800) Inbound Instructions
Instructions for the Inbound work flow. Free text.
Instructions_OutN, CstringVARCHAR2(800) Outbound Instructions
Instructions for the Outbound work flow. Free text.
