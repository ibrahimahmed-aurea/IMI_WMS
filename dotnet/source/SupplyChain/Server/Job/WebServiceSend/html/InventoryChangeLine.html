Segment InventoryChangeLine

Description:

Inventory Change
Changes of Item Loads that effect the Balance or Status Code in the Host System.

Attributes:

TagMethodTypeOracle TypeMandatoryNote
OPCODEN, C, RstringVARCHAR2(1) Operation Code
1 - New
EmployeeIdNstringVARCHAR2(35) User Identity
The identity of the User. This is the user that registered the change.
TimeStampNNullableDATE Queue Processing Time
Date and Time when the change has been processed in the Item Load Change Queue.
ItemLoadIdentityNstringVARCHAR2(35) Item Load Identity
The identity of the Item Load that has been changed.
OperationCodeNstringVARCHAR2(1) Operation Code
0 = No processing
1 = New
2 = Change
3 = Delete
9 = Mass processing
InventoryChangeCodeNstringVARCHAR2(2) Item Load Change Code
Code for the change that has been done for the Item Load.
These codes could be registered per installation and there also exist a couple of standards Codes.
The following Codes are reserved for the system (the Type that is connected by default is shown in parenthesis) :
AS = After Split (Q = Quantity / Balance)
BL = Hold/Release (B = Hold / Release Hold)
CR* = Correction (Q = Quantity / Balance)
NP = New Item Load from Pick Location (O = Other)
PC = Package Change (P = Package Exchange)
RA* = Correction of Receipt (Q = Quantity / Balance)
RF = RF Inventory Audit (Q = Quantity / Balance)
SC* = Scrapping (Q = Quantity / Balance)
ST = Inventory Audit (Q = Quantity / Balance)
TA = Transfer / Refine - Qty increased (Q = Quantity / Balance)
TC = Transfer / Refine - Qty decreased (Q = Quantity / Balance)
TR = Transfer Movement (O = Other)
(* = Only used when the Host System = IMI Order)
InventoryChangeTextNstringVARCHAR2(35) Item Load Change Text
